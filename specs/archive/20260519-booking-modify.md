# 歸檔記錄：[預約修改] 功能

- **日期**：2026-05-19
- **狀態**：✅ 已完成並驗收
- **負責人**：Gemini CLI

## 功能摘要
實作了顧客修改預約時間的功能，包含：
- **時段管理**：引入 `time_slots` 資料表，支援精確到服務項目的可用時段查詢。
- **改期邏輯 (RPC)**：使用 Supabase RPC 實作原子化的改期操作，確保資料一致性並檢查 24 小時限制。
- **前端互動**：開發了 `ModifyBookingModal` 組件，提供直觀的日期與時段選擇。

## 技術重點
- **原子性操作**：改期涉及釋放舊時段、佔用新時段與更新預約紀錄，均在一個資料庫事務 (RPC) 中完成。
- **時區處理**：統一使用 ISO 格式進行通訊，並在前端根據使用者本地時區進行格式化。
- **安全性**：透過 RLS 確保只有預約擁有者能發起改期請求。
